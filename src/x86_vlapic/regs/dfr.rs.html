20/// Destination Format Register using MMIO.
21/// - Address: FEE0 00E0H
22/// - Value after reset: FFFF FFFFH
23///
24/// The 4-bit model field in this register selects one of two models (flat or cluster) that can be used to interpret the MDA when using logical destination mode.
25/// **Flat Model** — This model is selected by programming DFR bits 28 through 31 to 1111.
26/// Here, a unique logical APIC ID can be established for up to 8 local APICs by setting a different bit in the logical APIC ID field of the LDR for each local APIC.
27/// A group of local APICs can then be selected by setting one or more bits in the MDA.
28///
29/// **Cluster Model** — This model is selected by programming DFR bits 28 through 31 to 0000.
30/// This model supports two basic destination schemes: flat cluster and hierarchical cluster.
31pub type DestinationFormatRegisterMmio = ReadWrite<u32, DESTINATION_FORMAT::Register>;
32
33/// A read-write copy of Destination Format Register (FEE0 00E0H).
34///
35/// This behaves very similarly to a MMIO read-write register, but instead of doing a
36/// volatile read to MMIO to get the value for each function call, a copy of the
37/// register contents are stored locally in memory.
38pub type DestinationFormatRegisterLocal = LocalRegisterCopy<u32, DESTINATION_FORMAT::Register>;
